해설: 동일 콘텐츠에 소제목(H2/H3)과 명확한 정의 문장 구조를 적용했을 때, Perplexity의 인용 빈도가 평균 2.3배 증가하는 패턴이 관찰됐습니다.

1) 브랜드명을 문장 마지막에 배치

4) 브랜드명 대신 약어를 사용

[Schema Markup이 AI 검색 인용에 미치는 영향 (Version A~E)]

Q1. Schema Markup이란 무엇인가요?
1) 웹사이트 방문자 행동을 추적하는 분석 도구
2) AI와 검색엔진이 웹페이지의 내용을 구조적으로 이해하도록 돕는 코드 (구조화 데이터)
3) 이미지 최적화를 위한 압축 알고리즘
4) 서버 간 통신을 암호화하는 보안 프로토콜
정답: 2
해설: Schema Markup은 HTML에 추가하는 구조화 데이터로, AI와 검색엔진이 페이지가 상품인지, 리뷰인지, FAQ인지를 명확히 파악할 수 있게 해줍니다.

Q2. Schema Markup의 권장 삽입 형식은?
1) XML
2) CSV
3) JSON-LD
4) YAML
정답: 3
해설: Schema Markup을 삽입하는 방식은 JSON-LD, Microdata, RDFa 세 가지가 있으며, 현재는 JSON-LD가 가장 널리 권장됩니다.

Q3. GEO 관점에서 우선 적용을 권장하는 Schema 유형 2가지는?
1) Image Schema와 Video Schema
2) Product Schema와 Review Schema
3) FAQ Schema와 Article Schema
4) Person Schema와 Event Schema
정답: 3
해설: GEO 관점에서 FAQ Schema(질문-답변 구조 명시)와 Article Schema(콘텐츠 신뢰도와 발행 맥락 구조화)를 우선 적용하는 것이 권장됩니다.

Q4. Schema Markup에 대한 올바른 이해는?
1) Schema Markup을 적용하면 AI 인용이 반드시 보장된다
2) Schema Markup 없이는 AI에 절대 인용될 수 없다
3) Schema Markup은 AI 인용을 보장하지 않지만, AI가 콘텐츠를 정확히 해석하는 데 유리한 조건을 만들어준다
4) Schema Markup은 구글에만 적용되고 ChatGPT에는 영향이 없다
정답: 3
해설: Schema Markup은 AI 인용을 보장하지 않습니다. 그러나 AI가 콘텐츠를 정확히 해석하고 인용 후보로 인식하는 데 유리한 조건을 만들어줍니다.

Q5. Schema Markup 적용 시 주의해야 할 사항은?
1) Schema는 한 번 적용하면 업데이트할 필요가 없다
2) 리뷰 수, 평점, 가격 등 수치는 실제 데이터와 반드시 일치시켜야 한다
3) Schema는 페이지 하단에만 삽입해야 한다
4) Schema는 모바일 버전에는 적용하지 않아도 된다
정답: 2
해설: 리뷰 수, 평점, 가격 등 수치가 실제 데이터와 다르게 하드코딩되어 있으면 신뢰도 문제로 이어질 수 있어 반드시 실제 데이터와 일치시켜야 합니다.
